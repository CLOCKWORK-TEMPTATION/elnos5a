# أفان تيتر | Avan Titre
[![FOSSA Status](https://app.fossa.com/api/projects/git%2Bgithub.com%2FCLOCKWORK-TEMPTATION%2Felnos5a.svg?type=shield)](https://app.fossa.com/projects/git%2Bgithub.com%2FCLOCKWORK-TEMPTATION%2Felnos5a?ref=badge_shield)


**محرر سيناريو عربي احترافي للويب**

محرر سيناريو عربي مبني على Tiptap v3 مع نظام تصنيف ذكي من 4 طبقات يدعم 8 أنواع عناصر سينمائية.

---

## نظرة عامة

أفان تيتر هو محرر سيناريو عربي احترافي يعمل على الويب، يهدف إلى مساعدة الكتّاب العرب في كتابة سيناريوهاتهم بسهولة ودقة. يعتمد المشروع على تقنيات حديثة تشمل Next.js 15 وReact 19 وTypeScript، مع نظام تصنيف تلقائي ذكي يحدد نوع كل سطر في النص (مش动作، حوار، شخصية، انتقال، إلخ).

يتميز المشروع بنظام تصنيفي هجين يعمل على 4 طبقات متتالية: أنماط Regex الأساسية، قواعد السياق، التصنيف الهجين، ثم وكيل AI كطبقة أخيرة للحالات الصعبة. هذا التصميم يضمن دقة عالية في التصنيف مع سرعة معالجة تصل إلى 1000 سطر في الثانية.

## الميزات الرئيسية

### المحرر الذكي

يعتمد المحرر على Tiptap v3 (المبني على ProseMirror) لتوفير تجربة كتابة سلسة تدعم الكتابة العربية من اليمين لليسار. يتضمن المحرر ميزات متعددة تشمل: التنسيق التلقائي للعناصر السينمائية، الإكمال التلقائي لأسماء الشخصيات، تصحيح الأخطاء الشائعة في النصوص السينمائية، وتتبع السياق عبر Document Context Graph.

يدعم المحرر 8 أنواع عناصر أساسية: البسملة (basmala)، رأس المشهد بأنواعه الثلاثة، الحركة (action)، الشخصية (character)، الحوار (dialogue)، الملاحظة الجانبية (parenthetical)، والانتقال (transition). كل نوع له تنسيق خاص به يتبع معايير الصناعة السينمائية.

### نظام التصنيف الرباعي

يعمل نظام التصنيف على أربع طبقات متتالية لضمان أعلى دقة ممكنة. الطبقة الأولى تستخدم أنماط Regex سريعة لتحديد الأنماط الواضحة. الطبقة الثانية تستخدم قواعد السياق لتتبع تسلسل العناصر وتصحيح الأخطاء المبنية على السياق. الطبقة الثالثة تجمع بين الطريقتين باستخدام scoring وتقييم ثقة. الطبقة الرابعة والأخيرة تستخدم Claude AI للتعامل مع الحالات الصعبة التي لم تستطع الطبقات السابقة تصنيفها بدقة.

### استيراد الملفات

يدعم المشروع استيراد الملفات بصيغ متعددة تشمل: PDF عبر Mistral AI OCR، DOC وDOCX عبر Antiword وMammoth، ونصوص عادية بصيغة Fountain. يتضمن نظام الاستيراد خطوط معالجة متعددة تشمل: التنظيف، التطبيع، التحقق من الجودة، تصحيح الأخطاء، ثم الإدراج في المحرر.

### الذكاء الاصطناعي

يتكامل المشروع مع عدة مزودين للذكاء الاصطناعي: Anthropic Claude للمراجعة والتصحيح، Mistral AI لاستخراج النصوص من PDF، Google Gemini لتحسين السياق وRAG، وMoonshot Kimi لحل حالات الشك في التصنيف.

## التثبيت السريع

### المتطلبات

تأكد من توفر المتطلبات التالية قبل التثبيت: Node.js الإصدار 20 أو أحدث، وpnpm الإصدار 10 أو أحدث، وGit للتعامل مع المستودع.

### الخطوات

استنسخ المستودع على جهازك باستخدام الأمر `git clone` مع رابط المستودع المناسب. بعد الاستنساخ، ثبت التبعيات عبر الأمر `pnpm install` في مجلد المشروع الرئيسي. ثم شغّل خادم التطوير الخلفي للأمام via الأمر `pnpm run dev` الذي يشغّل تطبيق Next.js وخادم Express على المنافذ المناسبة.

للتحقق من التثبيت، افتح المتصفح وانتقل إلى العنوان `http://localhost:3000` لمشاهدة المحرر. إذا واجهت أي مشاكل، راجع قسم استكشاف الأخطاء في هذا المستدفق.

## الاستخدام الأساسي

### الكتابة في المحرر

ابدأ بكتابة النص العربي في منطقة المحرر الرئيسية. سيقوم النظام تلقائياً بتصنيف كل سطر عند الضغط على Enter أو عند لصق نص جديد. يمكنك رؤية نوع كل عنصر من خلال اللون والتنسيق المطبق عليه في المحرر.

للحصول على أفضل النتائج، اتبع تسلسلاً منطقياً في كتابة المشاهد: ابدأ برأس المشهد، تبعه بوصف الحركة، ثم أضف أسماء الشخصيات بخط عريض، واكتب الحوار بعد اسم الشخصية مباشرة.

### استيراد ملف موجود

انقر على زر الاستيراد في شريط الأدوات العلوي. اختر الملف المطلوب من جهازك (PDF، DOCX، أو نص عادي). سيقوم النظام بمعالجة الملف وعرضه في المحرر مع تصنيف تلقائي لجميع العناصر.

### التصدير

يدعم المشروع تصدير السيناريو بعدة صيغ: PDF للتنسيق النهائي، DOCX للتعديل في Word، Fountain للتنسيق القياسي السينمائي، وFDX لصيغة Final Draft.

## أوامر التطوير

### تشغيل المشروع

للتطوير العام، استخدم `pnpm run dev` لتشغيل التطبيق مع الخادم الخلفي. للتطوير الكامل مع جميع الخدمات، استخدم `pnpm run start` التي تشغّل مرحلة preflight ثم وضع التطوير.

### الاختبارات

يوفر المشروع عدة مستويات من الاختبارات: `pnpm run test` للاختبارات الوحدة والتكامل مع التغطية، `pnpm run test:unit` للاختبارات الوحدة فقط، `pnpm run test:integration` لاختبارات التكامل، و`pnpm run test:e2e` لاختبارات end-to-end عبر Playwright.

### بناء المشروع

لبناء نسخة الإنتاج، استخدم `pnpm run build` التي تنشئ تطبيق Next.js الأمثل. لمعاينة البناء، استخدم `pnpm run preview`.

### أدوات إضافية

يتضمن المشروع أدوات إضافية تشمل: `pnpm run lint` لفحص جودة الكود، `pnpm run format` لتنسيق الكود تلقائياً، `pnpm run typecheck` لفحص الأنواع TypeScript.

## هيكل المشروع

### المجلدات الرئيسية

يحتوي المشروع على عدة مجلدات رئيسية: المجلد `app/` يتضمن ملفات Next.js App Router، المجلد `src/` يحتوي الكود المصدري الرئيسي، المجلد `server/` للخادم الخلفي Express، ومجلد `tests/` لاختبارات الوحدة والتكامل والنهاية-إلى-النهاية.

### الكود المصدري

يمثل المجلد `src/extensions/` جوهر المشروع بـ 38 ملفاً لمحرك التصنيف ومكونات Tiptap. المجلد `src/pipeline/` يحتوي نظام الاستيراد والمعالجة. المجلد `src/components/` يضم مكونات واجهة المستخدم. المجلد `src/ocr-arabic-pdf-to-txt-pipeline/` لنظام OCR منفصل. ومجلد `src/rag/` لنظام Qdrant للبحث الدلالي.

### الخادم الخلفي

يقع الخادم الخلفي في مجلد `server/` الذي يحتوي 28 ملفاً بامتداد mjs. يعمل الخادم على المنفذ 8787 ويتعامل مع استخراج الملفات ومعالجة OCR وخدمات الذكاء الاصطناعي.

## المعمارية

### النمط الهجين

يعتمد المشروع على نمط معماري هجين يجمع بين ثلاثة أنماط: Next.js App Router للتوجيه وServer-Side Rendering، React Components لإدارة حالة واجهة المستخدم، وImperative Classes لدورة حياة Tiptap وProseMirror.

هذا التصميم يهدف لتجنب مشاكل المزامنة بين React وProseMirror التي تحدث عند استخدام حالات React مباشرة مع المحرر.

### مصنع DOM

يستخدم المشروع نمط DOM Factory لبناء مكونات واجهة المستخدم بدلاً من JSX التقليدي. هذا يوفر تحكماً أدق في دورة حياة المكونات خاصة مع مكتبة Radix UI المستخدمة.

## الاختبارات

### استراتيجية الاختبار

يتبع المشروع استراتيجية اختبار متعددة المستويات: اختبارات الوحدة لعزل الدوال والمكونات، اختبارات التكامل لاختبار التفاعل بين الوحدات، واختبارات end-to-end لمحاكاة تجربة المستخدم الكاملة.

### أطر الاختبار

يستخدم المشروع Vitest للاختبارات الوحدة والتكامل مع بيئة jsdom للمحاكاة. لـ E2E، يستخدم Playwright مع متصفح Chromium. التغطية من خلالprovider v8 من Vitest.

### البناء للإنتاج

لتشغيل البناء، استخدم الأمر `pnpm run build` الذي ينشئ تطبيق Next.js محسنا جاهزاً للنشر. البناء يتضمن تحسينات للأداء وتشغيل جميع فحوصات الجودة.
